# P4.5 dogfood: Kubernetes ReplicaSet + ResourceQuota

**Date:** 2026-06-16

The P4.5 gate (research/26) validates the P4 workbench on a model larger than a
tutorial — not a new language feature. Domain: a Kubernetes ReplicaSet reconciling
Pods under a namespace `count/pods` ResourceQuota (`examples/k8s_replicaset/`). It
was chosen because its important guarantees are **safety / reachability**, while
its headline "eventually converges" is **liveness** — deliberately out of scope
(research/25). It exercises the full advanced surface at once: bounded multiplicity
(`Scope`), presence (`Create`/`Delete`), `Count` aggregates, `ownerReference`
provenance, and `NoDeadEnd`/`DeadActions`.

## Final model

Two ReplicaSets competing for one quota, 3 Pod slots, bare (unowned) pods, orphan
acquisition. **1792 reachable states, 13 008 edges, 4 roots, max depth 12, no dead
ends.** Six properties, all green and all framed without liveness:

- `quota_never_exceeded` (AlwaysHolds) — the live pod count never exceeds the quota.
- `both_converged_is_reachable`, `quota_can_starve_a_replicaset`,
  `bare_pods_can_starve_a_replicaset` (Reachable).
- `rs0_converged_always_recoverable` (NoDeadEnd) — a converged count is reachable
  from *every* reachable state; recovery is always possible, not inevitable.
- `every_action_usable` (DeadActions) — every action, including orphan acquisition,
  is reachable.

## Change series (measured)

Each change is a real requirement, applied on top of the previous, kept green.

| step | requirement change | model diff | states | edges |
|---|---|---:|---:|---:|
| 1 | core: 1 ReplicaSet reconciles Pods under the quota | new | 80 | 408 |
| 2 | a **second** competing ReplicaSet (per-owner `Count`, where-map) | +73 / −51 | 864 | 4 896 |
| 3 | **bare pods** consume the quota (create/delete) | +24 | 1 792 | 11 520 |
| 4 | **orphan acquisition** (a controller adopts an unowned pod) + DeadActions | +18 | 1 792 | 13 008 |

Observations:

- The second ReplicaSet was the only structural change (≈11× states); bare pods
  doubled the state space (pods reach owner `NONE`); acquisition added transitions,
  not states (it relabels an existing pod), so states held at 1792 while edges grew.
- Every safety/reachability property survived each change. Adding bare pods did
  **not** create a dead end because `delete_bare_pod` frees the quota — the
  `NoDeadEnd` recovery property kept holding, which is exactly the kind of
  regression a story-shaped example would miss.

## Using the workbench on itself

- `analint affects Pod.owner` pinpoints the writers (the `acquire_orphan` bindings)
  before a change — the impact radius without opening files.
- `analint explore` gives the summary-first shape (roots/states/edges/depth,
  branching, fired/excluded) and the compact JSON projection.
- `analint trace bare_pods_can_starve_a_replicaset` reads the witness as states and
  changes: `init #2 (quota=1) → create_bare_pod(p0) → scale_up(rs0)` ⇒ a bare pod
  holds the quota while rs0 wants one — a reviewable, domain-readable counterexample.

## Friction (the honest evidence)

1. **Scope slots default to *present*.** A "namespace starts empty" model needs an
   explicit `Absent(...)` initial, and `Initial` requires a `vary`, so the starting
   quota is varied over its domain (a natural multi-root) rather than a degenerate
   clause. A first-class "empty scope" initial would be cleaner.
2. **Per-owner counting needs an identity where-map.** `Count(pod, pod.owner ==
   rs_owner)` parameterised by ReplicaSet uses the same `Implies(rs==slot, owner==…)`
   plumbing as OAuth's `code_id`. It works, but the rs↔ownerReference binding is
   boilerplate the DSL does not express directly.
3. **Some intuitive "invariants" are not invariants.** "Owned never exceeds desired"
   is false (a scale-down leaves a recoverable surplus — real k8s); the honest
   property is recoverability. "Live ≤ quota" only holds because we model quota
   decreases conservatively (not below current usage); real k8s allows lowering the
   limit below usage and merely blocks new pods. Both are documented abstractions.

## Verdict

The workbench handles a project-sized model (~1800 states, multiplicity + presence
+ aggregates + provenance) end to end: authoring, `affects`/`explore`/`trace`
diagnostics, and a measured change series with witnesses. **No new primitive was
demanded** — `Count`, presence, `NoDeadEnd` and `DeadActions` sufficed. The only
real gaps are ergonomic (empty-scope initials; an identity/ownership binding), and
the reachability-only framing kept every claim honest: liveness (eventual
convergence) was excluded by design, not silently weakened. P4.5 passes.
