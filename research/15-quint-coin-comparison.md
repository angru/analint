# P6: перевод Quint coin.qnt в analint — прямое сравнение

Эксперимент из research/14 §P6: взять *чужую* реальную спецификацию,
перевести и честно посмотреть, что будет. Выбран флагманский туториал Quint —
[coin.qnt](https://github.com/informalsystems/quint/blob/main/examples/tutorials/coin.qnt)
(модель Solidity-субвалюты: минтер, балансы, mint/send, инварианты
переполнения, тесты, темпоральное свойство). У него есть целый урок в
документации Quint — это их «лицо», не соломенное чучело.

Результат: `examples/coin/spec.py`.

## Метрики

| | Quint coin.qnt | analint examples/coin |
|---|---:|---:|
| Строк (с комментариями) | 209 | 208 |
| Деклараций действий | 2 параметризованных (+ step) | 9 конкретных (фабрика ×2) |
| Размер адресного пространства | 5 адресов, amount 0..2²⁵⁶−1 | 3 счёта, деноминация 1, баланс 0..5 |
| Проверка | Apalache (символьно) / quint run (рандомизированно) | исчерпывающий BFS, 216 состояний |
| Главная находка урока | totalSupply переполняется | воспроизведена: трасса `mint_to_alice ×5 → mint_to_bob ⇒ total_supply=6` |

Паритет по объёму при воспроизведении всех содержательных элементов:
mint/send с require-охранами, оба инварианта, темпоральное свойство,
три из четырёх тестов.

## Что перевелось 1:1

- `require(...)`-охраны → `pre=[...]` дословно (поэлементное соответствие
  закомментировано в коде).
- `temporal NoSupplyOverflow = always(inv)` → `AlwaysHolds(...)` — прямое
  соответствие, и наш вердикт информативнее: минимальный контрпример
  в доменных именах действий вместо дампа состояний.
- `run sendWithoutMintTest = init.then(send(...)).fail()` →
  `Scenario(..., expected=Expect.FAIL)` — наша запись читается лучше.
- `val balancesRangeInv = ADDR.forall(a => isUInt(balances.get(a)))` →
  **исчез как отдельный инвариант**: `Field(0, ge=0, le=MAX)` даёт диапазон
  сразу в трёх механизмах (конструирование, пост-состояние, границы движка).
  Один из двух инвариантов Quint у нас структурный. Очко analint.

## Где пришлось платить (наши границы, подтверждённые делом)

1. **Карта → ручная мультипликация.** `balances: Addr -> UInt` + одно
   параметризованное `send(sender, receiver, amount)` превратились в
   3 класса-счёта и 9 конкретных действий. Фабрика (host-language слой,
   легитимный по research/14 §3.6) сжала это до ~25 строк, но рост —
   квадратичный по адресам: 10 адресов = 90 действий. **Это главная
   разница языков**, теперь измеренная: P4 (multiplicity + параметризация)
   подтверждён как приоритет.
2. **Нет арифметического AST.** `totalSupply = fold(+)` невыразим —
   денормализованный счётчик `Ledger.total_supply`, поддерживаемый каждым
   mint. Работает, но это ровно тот вид ручной денормализации, из-за
   которой в реальных моделях появляются рассинхроны. Подтверждает
   P4-пункт «typed arithmetic/derived expressions».
3. **Нет цепочек.** `init.then(mint).then(send).then(assert)` — исполняемая
   последовательность; наш Scenario запускает одно действие, промежуточное
   состояние кодируется в `given`. Терпимо для тестов, но «исполняемый
   Flow» — очевидный кандидат в роадмап (он же model-based testing P3).
4. **Символьная проверка vs явная.** Quint честно проверяет `2²⁵⁶−1` через
   Apalache; мы сжали домен до 5. Для туториала неважно, для криптографии —
   важно. Наша ниша — конечные доменные модели, не big-int протоколы.
5. **Недетерминированная инициализация** (`nondet sender = oneOf(ADDR)` для
   минтера) → зафиксирована конструктивно (mint-действия с `by=Minter`).
   Потеря несущественна здесь, но `given`-вариативность запросов покрывает
   это лишь частично.

## Что analint добавил сверх оригинала

- **Три уровня проверки одним объявлением** (`Field`) — у Quint диапазон
  это инвариант, который нужно не забыть проверить.
- **Акторы**: `by=Minter` / `by=Holder` выражают msg.sender-роли структурно.
- **`Expect.FAIL`-сценарии** как первоклассные «заблокированные пути».
- **Интроспекция**: `analint affects Ledger.total_supply` мгновенно
  показывает все действия, трогающие supply; у Quint — grep.
- **Никакой новой грамматики**: перевод писался в обычном Python с
  автодополнением; порог входа — нулевой для Python-команды.

## Вывод эксперимента

Гипотеза позиционирования (research/14 §8.3) подтверждается предметно:
analint переводит флагманский пример Quint в сопоставимом объёме, с более
читаемыми тестами и вердиктами, **пока модель — конечная и синглтонная**;
стена начинается ровно там, где у Quint начинаются коллекции и параметры.
Приоритет P4 (арифметический AST → параметризация → bounded multiplicity)
из аудита подтверждён практикой, а не рассуждением.

Пример намеренно красный (как trollbridge): нарушение supply-инварианта —
это и есть урок Quint, воспроизведённый дословно.
